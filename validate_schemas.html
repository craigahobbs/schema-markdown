Validate Schemas¶

>>> import uuid
>>> types = schema_markdown.parse_schema_markdown('''
... struct MyStruct
...     int a
...     uuid b
... ''')
>>> schema_markdown.validate_type(types, 'MyStruct', {'a': 5, 'b': uuid.UUID('8252121c-7f4f-4b6d-a7e5-f42ca6fdb64c')})
{'a': 5, 'b': UUID('8252121c-7f4f-4b6d-a7e5-f42ca6fdb64c')}

>>> try:
...     schema_markdown.validate_type(types, 'MyStruct', {'a': 5, 'b': 7})
... except schema_markdown.ValidationError as exc:
...     str(exc)
"Invalid value 7 (type 'int') for member 'b', expected type 'uuid'"
schema_markdown.validate_type(types, type_name, value, member_fqn=None)¶

Type-validate a value using the schema-markdown user type model. Container values are duplicated since some member types are transformed during validation.

Parameters

  • types (dict) – The map of user type name to user type model

  • type_name (str) – The type name

  • value (object) – The value object to validate

  • member_fqn (str) – The fully-qualified member name

Returns

The validated, transformed value object

Raises

ValidationError – A validation error occurred
